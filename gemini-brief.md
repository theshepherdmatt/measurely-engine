## What Measurely Is

Measurely is a **browser-based acoustic analysis tool** for audio engineers and hi-fi listeners. It has two jobs:

1. **Show you your room in 3D** — a live Three.js render of your listening space, speakers, furniture and acoustic treatment
2. **Tell you what's wrong acoustically** — SBIR interference, standing waves, early reflections — visualised directly onto that 3D room

The user sets up their room via a sidebar (dimensions, speaker position, listener position, treatment). The 3D view responds in real time. A REW measurement (from `http://localhost:4735`) can be loaded to drive real acoustic analysis instead of simulation.

---

## The Tech Stack (what Gemini must not change)

| Layer | Technology |
|---|---|
| 3D engine | Three.js (vanilla, no bundler) |
| UI controls | `sidebar-component-library.js` — all controls generated by `SCL.renderXxxSection()` |
| Physics math | `acoustics.js` — `computeSbir()`, `computeRoomModes()` |
| Styling | Vanilla CSS, CSS custom properties (`--mly-*`) |
| Entry point | `demo.html` — single file, `<script type="module">` |
| REW API | `http://localhost:4735` — measurement source of truth |

**Golden rule: sidebar controls are monochrome. Colour only lives in the 3D viewport.**

## The Current UI — What the Screenshots Show

The app has **4 tabs** across the top:

- `01 SETUP` — room dimensions, speaker config, furniture
- `02 MEASURE` — REW bridge panel, sweep trigger, WAV file upload
- `03 ANALYSIS` — diagnostic toggle list + context panel with live sliders
- `04 TREAT` — treatment recommendations, panel colour swatches, Preview All

### What is working in the screenshots:

**Analysis tab:**
- A flat white diagnostic list with 4 rows: `Source & Listener`, `Boundary Interference`, `Standing Waves`, `Early Reflections`
- Each row: label (13px Inter) · acronym (10px Courier ghost) · SHOW pill (teal when active)
- Above the list: a **context panel** that mirrors the 2 key sliders — "Speakers from wall" (teal track) and "Listening position" (purple track) — with live +/- delta readout on drag
- Below the list: a "Simulation sound waves" toggle
- A lightbulb icon insight box with Purple-50 lavender background when a diagnostic is active

**Treatment tab:**
- A `RECOMMENDATIONS` section (populated after a sweep — currently shows placeholder text)
- A `VISUALISE` section with a "Preview All Recommendations" button and 8 colour swatches
- A `TREATMENT` section with SCL-rendered toggles: Bass traps, Wall panels, Side panels, Ceiling cloud

**3D viewport (always visible left of sidebar):**
- Pink `#FF107A` = SBIR interference / untreated state
- Teal `#00F5FF` / `#0D9488` = treated / active diagnostic
- Purple `#7C3AED` = listening position / insight colour
- Wave rings animate outward from speaker positions

## What Is NOT Yet Working (the bugs to fix)

1. **Clicking "Boundary Interference" in the Analysis tab does not activate the SBIR heatmap in the 3D view.** The row turns teal (UI state updates) but `focusIssue('sbir')` / the overlay activation is not firing. The click handler in `diagList` needs to be verified.

2. **The "Preview All Recommendations" button** updates `demoState.environment.treatment` correctly but `treatAPI?.setAllOn?.()` is a stub — the SCL doesn't expose that method yet, so the 3D room doesn't actually show ghost panels.

3. **Colour swatches** call `treatAPI?.setColour?.(hex)` — optional-chain means it silently fails if SCL doesn't expose `setColour`. Needs verification.
